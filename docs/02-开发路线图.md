# 开发路线图

## 技术路线

当前原型采用 React、TypeScript 和本地模拟数据。待交互验证后，再接入 SQLite 与目录扫描服务，避免过早固定后端结构。

```text
浏览器界面
  -> 资源索引 API
      -> SQLite 元数据与关系
      -> 预览缓存
      -> VS 只读连接器
      -> NAS 只读连接器
```

## 阶段安排

### 阶段 0：需求与原型基础

- 建立独立项目目录。
- 固化需求边界和术语。
- 准备可运行的本地网页骨架。

验收：项目可启动，需求和计划可在仓库中持续更新。

### 阶段 1：可视化资源库

- 资源卡片、搜索、分类筛选。
- 详情面板、关联文件、来源路径。
- 场景集合和组件关系示例。
- 上传命名规则预览。

验收：使用模拟数据完成“找资源 -> 看详情 -> 看来源 -> 生成规范名称”的流程。

### 阶段 2：本地索引服务

- 引入 SQLite 数据库。
- 扫描 `mock_vs` 与 `mock_nas` 目录。
- 记录文件指纹、更新时间和关联状态。
- 支持增量扫描，避免每次重建全部索引。

验收：放入新模拟文件后，扫描器可以生成或更新资源记录。

### 阶段 3：预览流水线

- JPG、PNG 直接生成多尺寸缩略图。
- PSD 提取合成预览；失败时显示明确状态。
- 3D 文件通过离线任务生成固定角度预览。
- 预览结果缓存并可重新生成。

验收：门户不打开源文件也能识别大部分模拟资产。

### 阶段 4：规范上传与关系维护

- 根据版本、类别、标题、作者生成名称和路径。
- 同名、漏字段和目录冲突检查。
- 场景组件由用户确认关联，可记录“可复用/临时/待判断”。

验收：上传者无法绕过必填字段，管理员可以追踪规则版本。

### 阶段 5：AI 辅助

- 建议标签、说明和相似资源。
- 给出重复风险，不自动删除或合并。
- 所有 AI 建议进入人工确认队列。

验收：建议具有来源和置信度，用户可接受、修改或拒绝。

## 第一阶段验收清单

- 桌面端和窄屏均可正常浏览。
- 搜索与分类筛选即时生效。
- 资源卡片可切换并展示对应详情。
- 场景集合、上传规则页面可操作。
- 项目构建通过，无类型错误。
