# 企业接入与权限方案

更新时间：2026-07-14

## 结论

“Team Foundation”和“共享盘”不是同一个系统，也不应做成同一个连接器。

| 企业来源 | 主要职责 | Atlas 的接入方式 |
| --- | --- | --- |
| Azure DevOps Server / 旧称 TFS | 项目、权限、历史与 TFVC 签入签出 | 使用公司地址、账号和 `TF.exe` 或获准的 API 读取元数据 |
| TFVC | 集中式版本控制；服务器保存历史与 changeset | 记录服务器路径、changeset、版本状态和按需获取入口 |
| NAS / Windows 共享盘 | 通过 SMB 提供多人可访问的文件目录 | 索引 UNC 路径、大小、修改时间和预览图，不复制源文件 |
| Atlas | 把分散来源组织为工作域、模块、版本和内容 | 保存索引、权限映射、预览与审计，不替代企业存储 |

Visual Studio 是 TFVC 的客户端之一，不等于服务器。公司口中的 “Team Foundation” 很可能是旧版 TFS、现在的 Azure DevOps Server，版本库类型可能是 TFVC，也可能是 Git，需要技术部门确认。

## 身份与权限

正式环境不应由用户手填职位决定权限。推荐顺序是：

1. 使用公司 Windows AD、Entra ID 或现有统一登录确认身份。
2. 同步姓名、邮箱、职位和所属组；职位只用于界面显示。
3. 用“用户组/角色 + 工作域权限”决定读取、上传、下载、评审和管理能力。
4. 前端隐藏无权入口，服务端再次校验；每次上传、下载和权限修改写入审计日志。

| 角色 | 默认能力 |
| --- | --- |
| 制作人员 | 浏览授权工作域；向授权模块和版本上传；提交同名文件的新版本 |
| 把控人员 | 制作人员能力；评审、批注、通过/退回；查看所辖组进度 |
| 项目管理员 | 新增工作域、模块、版本；配置流程；查看审计与异常 |
| 系统管理员 | 配置账号同步、连接器、服务账号和权限模板 |

当前原型已提供三种模拟身份，用来验证界面和上传范围。它不是正式登录系统；生产权限必须在服务端执行。

## 从 ArtHub DAM 借用什么

ArtHub 的制作人员实践强调搜索/筛选/连续预览、大文件上传、分享评审、同名同格式文件形成新版本，以及 DCC 工具接入。美术把控和项目管理实践进一步使用目录权限组、阶段文件夹、评审批注、关注通知和审计日志。

Atlas 采用：

- “工作域 -> 模块 -> 版本 -> 内容”目录与团队权限绑定。
- 同一内容继续提交时默认形成新版本，而不是覆盖历史。
- 全局搜索、类型/人员/状态筛选和游标分页预览。
- 上传、下载、评审、索引和权限修改审计。
- 后续增加评审批注、关注通知和 Photoshop/Maya/3ds Max 辅助入口。

Atlas 暂不采用：

- 不在线解析 PSD、Blend、FBX 等工程源文件。
- 不把 NAS/TFVC 全量同步到门户。
- 不让浏览器直接持有企业共享盘的高权限账号。

## 本机安全实验环境

本机验证无需先安装完整 Azure DevOps Server。先用仓库内目录验证索引、上传、按需读取和文件占用：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\local-lab\Setup-LocalLab.ps1
powershell -ExecutionPolicy Bypass -File .\tools\local-lab\Test-LocalLab.ps1
```

脚本会在 `lab-data/` 建立模拟 TFVC 工作区和 NAS 目录，不包含公司数据，也不会默认修改 Windows 共享设置。确实要测试 SMB 时，在管理员 PowerShell 中显式运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\local-lab\Setup-LocalLab.ps1 -EnableSmbShare
```

## 公司只读试接入

第一次接公司环境只做只读试点，技术部门提供：

- Azure DevOps Server/TFS Collection URL 和项目名称。
- 版本库类型；若为 TFVC，提供一个非敏感服务器路径和 workspace 规则。
- NAS 的测试 UNC 路径，例如 `\\server\share\atlas-pilot`。
- 测试账号、认证方式、允许调用的 API/命令和网络/VPN条件。
- 约 100 至 1,000 个脱敏样例文件及其真实目录规则。

只读探测命令：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\local-lab\Test-CompanyConnectors.ps1 `
  -CollectionUrl "http://tfs-server:8080/tfs/DefaultCollection" `
  -TfvcServerPath "$/ProjectAurora/Pilot" `
  -NasPath "\\server\share\atlas-pilot"
```

必须在试点验收、权限审查和备份策略确认后，才开启真实上传。
