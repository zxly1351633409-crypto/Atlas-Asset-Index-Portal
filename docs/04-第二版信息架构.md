# 第二版信息架构

## 核心导航

```text
工作域 -> 模块 -> 版本 -> 内容条目 -> 源文件
```

工作域是项目中的工作目标，例如场景、角色、文案需求、系统需求、插画需求、插画、UI、模型和材质贴图。它是可配置字典，不与文件扩展名绑定。

## 对象职责

- `WorkDomain`：左侧一级导航，区分需求、产出和生产资产。
- `ProjectModule`：业务模块，例如荒原地图、装备淬炼、V6.2 主视觉。
- `ModuleVersion`：模块在项目版本中的快照，具有当前、历史、制作中等状态。
- `ContentItem`：版本中的具体需求或产出条目。
- `SourceFile`：一个内容条目可获取的 JPG、PSD、3D 工程、文档等文件变体。
- `Relation`：连接需求、产出、生产源和复用目标。

## 模块默认行为

- 点击模块进入模块工作区，不使用弹窗承载主要导航。
- 默认自动选择模块的当前有效版本，避免无内容状态。
- 版本轴第一个入口为“总览”，用于查看版本历史、最近变更和关联关系。
- 选择具体版本后，内容区只显示该版本条目，并标记新增、修改、延续和待确认。

## 新增与关联

原“上传”改为“新增/关联内容”，根据当前工作域、模块和版本自动填充上下文：

1. 关联已有 VS/NAS 路径：只建立索引和预览，不搬运源文件。
2. 上传预览文件：补充缩略图或评审图。
3. 提交新的源文件：根据规则生成名称与目标路径。

## 按需获取

门户常态只读取元数据和轻量预览。用户在详情中选择 JPG、PSD、3D 工程或文档后，连接器才从 VS/NAS 获取对应源文件。
