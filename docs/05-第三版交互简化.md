# 第三版交互简化

## 目标

降低页面信息密度，让主要流程只保留四个动作：选择工作域、打开模块、切换版本、预览内容。

## 参考原则

- Apple 产品页面：一个视区强调一个主要对象，减少同时竞争的文字和按钮。
- OpenAI Codex：使用安静的中性色、明确层级和按需展开的辅助信息。
- Frame.io：缩略图用于浏览，全屏查看器用于审阅；预览文件和高清原文件分开处理。

## 交互规则

- 单击内容条目：选中并显示简要来源信息。
- 双击内容条目：打开沉浸式预览。
- 图片条目：在预览层缩放查看，高清 JPG、PSD、3D 工程按格式获取。
- 需求条目：根据内容类型实时渲染为可阅读文档或表格，原始 DOCX/XLSX 按需获取。
- `Esc`：关闭预览层。

## 界面减法

- 移除模块首页的大块统计数字。
- 模块卡片只保留封面、名称、当前版本、负责人和更新时间。
- 版本轴保持紧凑，但使用明确的时间基线、日期和版本节点表达版本推进关系。
- 颜色以黑、白、浅灰为主，蓝色只表示当前选择，琥珀色只表示变更状态。
