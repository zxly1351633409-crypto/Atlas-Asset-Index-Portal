## 回溯规则

操作变更轴记录上传、修改、删除和回溯。它与项目版本轴分工不同：项目版本轴回答“当前查看哪个交付版本”，操作变更轴回答“这个内容经历过什么”。

1. 上传或修改都会创建操作记录和内容快照。
2. 删除只把内容标记为 `deleted`，不删除操作记录和快照。
3. 选择历史节点回溯时，系统复制该节点的文件清单并创建一个新修订。
4. 回溯本身也写入操作记录，因此可以继续审计和再次回溯。
5. 当前界面原型在本次浏览器会话中保留新增回溯；正式持久化目标已落入 `asset_operations`、`asset_snapshots` 和 `content_items.lifecycle_status`。

## 全工作域目录

门户识别以下根目录：

```text
ProjectAurora\Scenes                         -> 场景
ProjectAurora\Characters                     -> 角色
ProjectAurora\Illustrations                  -> 插画
ProjectAurora\UI                             -> UI
ProjectAurora\Requirements\Writing           -> 文案需求
ProjectAurora\Requirements\System            -> 系统需求
ProjectAurora\Requirements\Illustration      -> 插画需求
ProjectAurora\Models                         -> 模型
ProjectAurora\Materials                      -> 材质贴图
```

每个工作域使用相同层级，其中 `<模块>` 和 `<版本>` 可以先创建为空目录，门户也会显示。文件支持两种放法：

```text
# 简单内容：文件名直接成为内容名
F:\Atlas-Company-Share\ProjectAurora\<工作域>\<模块>\<版本>\成图.jpg

# 成套内容：把预览、源文件和导出归为一条内容
F:\Atlas-Company-Share\ProjectAurora\<工作域>\<模块>\<版本>\<内容名>\
  Preview\*.jpg|png|webp
  Source\*.psd|blend|ma|max|c4d
  Export\*.fbx|obj|gltf|jpg|png
  Documents\*.docx|xlsx
  <内容名>.asset.json
```

同名的 JPG、PSD 等直接文件会自动归为一条内容。正式视图只读取共享盘，不再显示荒原地图、主城活动地图等演示模块。门户只复制小型预览到本地缓存，并在清单中记录源文件真实路径、大小和来源。PSD、模型工程和高清文件不会被预加载。直接修改文件时使用文件名、大小和更新时间生成轻量指纹；新增和修改会进入内容的操作变更轴，删除记录保留在本机扫描状态中。

常用命令：

```powershell
# 首次建立九个工作域目录并扫描
powershell -ExecutionPolicy Bypass -File .\tools\local-lab\Initialize-ExternalContentShare.ps1

# 后台监视新增、修改和删除
powershell -ExecutionPolicy Bypass -File .\tools\local-lab\Start-ExternalContentShareWatcher.ps1

# 手动重建索引
powershell -ExecutionPolicy Bypass -File .\tools\local-lab\Scan-ExternalContentShare.ps1

# 自动验证场景内容的新增、修改和删除；测试结束后会清理临时内容
powershell -ExecutionPolicy Bypass -File .\tools\local-lab\Test-ExternalContentSync.ps1

# 停止后台监视
powershell -ExecutionPolicy Bypass -File .\tools\local-lab\Stop-ExternalContentShareWatcher.ps1
```

`Enable-ExternalModelSmbShare.ps1` 需要管理员权限。更换电脑后 UNC 主机名会变化，不应把当前电脑名当作公司正式配置。

## TFVC workspace

Visual Studio 和 Team Explorer 是客户端，不能仅凭本机文件夹模拟真实 TFVC 历史。连接公司环境还需要技术部门提供：

- Azure DevOps Server/TFS Collection URL。
- 以 `$/` 开头的非敏感 TFVC 试点路径。
- 有权读取该路径的公司账号及 VPN/网络条件。
- workspace 命名、签出锁和签入策略。

先运行预览模式确认客户端和目标本地路径，不会连接服务器，也不会下载文件：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\local-lab\Configure-TfvcWorkspace.ps1
```

取得公司参数后再显式应用映射；默认仍不执行 `tf get`：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\local-lab\Configure-TfvcWorkspace.ps1 `
  -CollectionUrl "http://tfs-server:8080/tfs/DefaultCollection" `
  -TfvcServerPath "$/ProjectAurora/Pilot" `
  -LocalPath "F:\Atlas-TFVC-Workspace\ProjectAurora" `
  -Apply
```

只有明确需要拉取试点内容时才添加 `-GetLatest`。正式接入优先做只读目录与 changeset 探测，确认权限、审计和备份后再开放上传或签入。

## 本机验收

1. 在任一工作域根目录下新增 `<模块>\<版本>\<内容名>`，并放入预览或源文件。
2. 等待监视器扫描后打开对应工作域，确认出现带“共享盘”标识的真实模块。
3. 进入该模块与版本，选择内容并核对源文件 UNC 路径和“操作变更轴”。
4. 选择非当前节点并执行回溯，确认轴首端出现新的“回溯”修订。
5. 关闭并重新打开操作变更轴，确认该修订在本次会话中仍存在。
