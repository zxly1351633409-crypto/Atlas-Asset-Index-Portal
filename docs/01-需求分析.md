# VS 资产门户：需求分析

## 1. 项目目标

在个人电脑上使用模拟数据，复现一套适合游戏美术团队的资源管理框架。项目不接触、不复制公司资源，只验证信息架构、交互流程和技术方案。

产品定位是“资源索引与预览门户”，而不是替代 Visual Studio、NAS 或版本管理系统：

- 原始文件继续保存在原有存储位置。
- 门户保存预览图、标签、版本、作者、分类、来源路径和文件关联。
- 使用者先在网页中找到资源，再按权限打开或下载源文件。
- 上传者通过向导生成统一的文件名、目录和元数据。

## 2. 当前核心问题

1. Visual Studio 主要展示文件名，无法快速判断图片、PSD 和场景内容。
2. VS 与 NAS 中的资源彼此分散，同一资产的 JPG、PSD、模型工程缺少关联。
3. 各组上传命名和目录习惯不一致，资源越多，检索成本越高。
4. 场景中的模型、材质和临时几何体没有清晰的复用关系。
5. AI 可以辅助打标签和发现相似内容，但不能独立决定资源是否可复用。

## 3. 产品对象

- `Asset`：用户看到的一条资源记录，例如场景图、角色图、模型或贴图。
- `SourceFile`：JPG、PSD、Blend、FBX 等实际文件及其来源路径。
- `Preview`：网页可直接查看的缩略图、高清图或转码结果。
- `Collection`：一个场景或主题下的一组资产。
- `ComponentLink`：场景与树、石头、建筑、材质等组件的人工确认关系。
- `UploadRule`：版本、分类、作者、原图/差分等命名与目录规则。
- `Tag`：人工标签与 AI 建议标签，二者必须区分来源。

## 4. 第一阶段边界

第一阶段使用本地模拟数据，优先验证以下闭环：

1. 浏览资源预览。
2. 按名称、标签、类型和业务分类筛选。
3. 查看资产详情及关联文件。
4. 查看场景集合与可复用组件关系。
5. 在上传向导中生成标准名称和推荐目录。

暂不在第一阶段实现真实公司账号、真实 VS/NAS 接入、PSD 在线分层解析、3D 在线编辑和 AI 自动判定复用。

## 5. 关键原则

- 索引优先：先建立可搜索目录，再考虑搬运大文件。
- 人工确认：AI 只生成建议，不直接修改分类和复用关系。
- 渐进接入：先扫描模拟目录，再替换为只读连接器。
- 可追溯：每条记录保留来源、版本、作者和更新时间。
- 规则前置：上传时完成规范校验，不把整理压力留到事后。
